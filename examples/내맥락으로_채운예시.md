# 예시: 빈 템플릿 → 내 맥락으로 채우기

> 핵심 메시지의 실증. 같은 골격이라도 **내 프로젝트 맥락**이 들어가야 결과가 산다.
> (아래는 내 실제 진행 중 프로젝트 기반 예시 — 모닝 마켓 브리핑, 영상 제작)

---

## 예시 1 — 모닝 마켓 브리핑 (작업지시 템플릿 채움)

```xml
<role>
너는 미국 증시 데이터를 한국어 모닝 브리핑으로 요약하는 금융 에디터다.
</role>

<context>
이 브리핑은 매일 아침 텔레그램 봇으로 자동 발송된다. 출근 전 1분 안에
어젯밤 미국장 핵심을 파악하는 게 목적이다. 월 비용 0원 유지가 전제다.
</context>

<task>
어젯밤 미국 3대 지수·주요 종목·환율 데이터를 받아 텔레그램용 브리핑을 생성해라.
</task>

<requirements>
1. 지수(S&P500/나스닥/다우) 등락률을 먼저, 그다음 특징주 3개
2. 텔레그램 메시지 형식 — 이모지 헤더 + 짧은 문단, 표 금지
3. 숫자는 모두 출처 데이터에서만. 추측 금지
</requirements>

<constraints>
- 유료 API 금지. 규칙 기반·무료 데이터 우선.
- 오버엔지니어링 금지. 발송에 필요한 것만.
</constraints>

<success_criteria>
텔레그램에서 그대로 읽혀 출근 전 30초에 핵심이 잡히고, 모든 수치에 근거가 있다.
</success_criteria>

끝내기 전에 success_criteria에 비춰 검증하고, 수치 근거가 빠졌으면 고쳐라.
```

**왜 이게 사는가:** "마켓 브리핑 써줘"는 일반론이 나온다. 위처럼 *텔레그램·1분·0원·규칙기반*이라는
내 제약을 넣으면 발송 가능한 결과가 한 번에 나온다.

---

## 예시 2 — 영상 제작 (행동 명시 + 엔진 맥락)

```xml
<context>
"영상 만들어줘"는 내 환경에서 Auto 패럴랙스 엔진(움직이는 영상) 기준이다.
my-video 정적 템플릿이 아니다.
</context>

<task>
{{주제}}로 Auto 패럴랙스 엔진을 써서 움직이는 영상을 생성해라.
제안만 하지 말고 실제로 생성까지 진행해라.
</task>

<success_criteria>
패럴랙스 모션이 적용된 영상 파일이 실제로 출력된다.
</success_criteria>
```

**왜 이게 사는가:** 엔진 선택이라는 *내 환경의 암묵 규칙*을 맥락으로 박아두면
매번 "어떤 엔진?"을 되묻지 않는다. + "제안 말고 실행"으로 도구 사용을 명시.

---

## 교훈

| 일반 템플릿 | 내 맥락을 채운 것 |
|---|---|
| "분석해줘" | "텔레그램용·1분·0원 제약으로 분석해줘" |
| "영상 만들어줘" | "Auto 패럴랙스 엔진으로, 실행까지" |
| "좋은 답 줘" | "success_criteria: 모든 수치에 근거" |

> 시중 프롬프트는 **골격**으로만 가져온다. 결과를 가르는 건 빈칸에 넣는 내 성공 기준·제약·예시다.
