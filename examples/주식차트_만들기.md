# 예시: "주식 차트 만들어줘"를 제대로 된 프롬프트로

> 가장 흔한 일반론 요청을 골격에 맞춰 채운 실전 예시.
> 포인트는 **빈칸을 내 맥락(데이터 출처·기술스택·디자인 기준·검증 방법)으로 채우는 것**.

---

## ❌ 일반론 (이렇게 주면 안 나온다)

```
주식 차트 만들어줘
```

→ 무슨 종목? 캔들? 라인? 데이터는 어디서? 웹? 파이썬? 정적 이미지? 인터랙티브?
모델이 전부 추측한다 → 제네릭한 결과 + 되묻기 반복.

---

## ✅ 채운 버전 (작업지시 템플릿 기반)

```xml
<role>
너는 인터랙티브 금융 차트를 만드는 프론트엔드 + 데이터 엔지니어다.
</role>

<context>
이 차트는 내 종목 모니터링용 대시보드에 들어간다. 매일 장 마감 후 데이터를
갱신해 추세를 한눈에 보는 게 목적이다. 유료 데이터 API는 쓸 수 없다(과금 이력 있음).
</context>

<task>
지정한 종목의 일봉 캔들스틱 차트를 인터랙티브 웹 차트로 구현해라.
설명만 하지 말고 실행 가능한 코드까지 완성해라.
</task>

<requirements>
1. 데이터: 무료 소스에서 OHLCV를 가져온다 (예: yfinance / stooq CSV). 키 필요한 유료 API 금지.
2. 차트: 캔들스틱 + 거래량 서브차트 + 20/60일 이동평균선. 줌·팬·툴팁 인터랙션.
3. 스택: 가벼운 라이브러리 사용 (Lightweight Charts 또는 ECharts). 차트 JS는 동적 import.
4. 형식: 단일 진입점에서 종목 코드만 바꾸면 재사용되도록 함수로 분리.
5. 에러: 데이터 빈 응답·네트워크 실패를 명시적으로 처리하고 사용자 메시지를 띄운다.
</requirements>

<design>
제네릭한 "AI slop" 금지. 의도된 금융 대시보드처럼 보이게:
- 상승/하락 색을 시맨틱하게(빨강↑/파랑↓ 한국 관습 or 초록/빨강 — 하나로 일관)
- 다크 배경 + 차분한 그리드, 가격축 우측 정렬
- 호버 시 십자선(crosshair) + OHLC 값 표시
</design>

<constraints>
- 오버엔지니어링 금지. 캔들·거래량·MA·인터랙션까지만. 매매신호·백테스트 등은 요청 전까지 만들지 마라.
- 이미지 width/height 명시, 차트 JS는 지연 로드 (성능 예산 지킴).
</constraints>

<success_criteria>
브라우저에서 열면 종목의 일봉 캔들·거래량·이동평균이 그려지고,
줌/팬/툴팁이 작동하며, 데이터가 없을 때 깨지지 않고 안내 메시지가 뜬다.
</success_criteria>

끝내기 전에 success_criteria의 각 항목을 직접 점검하고(빈 데이터 케이스 포함),
안 되는 게 있으면 고친 뒤 완료라고 말해라.
```

---

## 왜 각 블록이 필요한가

| 블록 | 없으면 생기는 일 |
|---|---|
| `context` (모니터링·마감후·유료금지) | 데이터 소스를 유료 API로 짜버림 → 과금 |
| `requirements 1` (무료 OHLCV) | API 키 요구하는 코드가 나옴 |
| `requirements 2` (캔들+거래량+MA) | 밋밋한 라인 차트만 나옴 |
| `design` (anti-slop) | 보라색 그라데이션의 제네릭 차트 |
| `constraints` (오버엔지니어링 금지) | 안 시킨 백테스트·매매신호까지 붙임 |
| `success_criteria` + 자기검증 | 빈 데이터에서 깨지는 채 "완성" 선언 |

> **교훈:** "주식 차트"라는 한 단어 요청도, *데이터 출처·차트 종류·스택·디자인·검증*만
> 박아두면 한 번에 쓸 수 있는 결과가 나온다. 이게 골격에 내 맥락을 채운다는 뜻.

---

## 변형 — 파이썬 정적 차트가 필요할 때

웹이 아니라 리포트용 이미지라면 `requirements`만 바꾸면 된다:

```
3. 스택: matplotlib + mplfinance. PNG로 저장 (width/height·dpi 명시).
4. 형식: generate_chart(ticker, start, end) -> 저장경로 반환 함수.
```

골격(role/context/task/constraints/success_criteria)은 그대로, **요구사항만 교체**.
