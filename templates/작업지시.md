# 템플릿: 작업 지시

> 코드/구현/작업을 시킬 때. <code v-pre>{{ }}</code>를 내 맥락으로 채운다.

```xml
<role>
{{역할 한 문장 — 예: 너는 파이썬 자동화 전문가다}}
</role>

<context>
{{왜 이 작업이 필요한지 / 어디에 쓰이는지 — 이유를 줘야 일반화한다}}
</context>

<task>
{{무엇을 할지 한 문장으로 명확히. "제안"이 아니라 "구현/수정"으로}}
</task>

<requirements>
1. {{순서·완전성이 중요한 단계를 번호로}}
2. {{출력 형식·제약 구체적으로}}
3. {{하지 말 것이 아니라 "이렇게 해"로}}
</requirements>

<constraints>
- 오버엔지니어링 금지. 요청한 것·명백히 필요한 것만.
- {{프로젝트별 제약 — 예: 유료 API 금지, 규칙 기반 우선}}
</constraints>

<success_criteria>
{{이게 되면 성공이다 — 검증 가능한 형태로}}
</success_criteria>

끝내기 전에 위 success_criteria에 비춰 결과를 스스로 검증하고, 안 맞으면 고쳐라.
```
