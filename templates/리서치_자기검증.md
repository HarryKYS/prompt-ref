# 템플릿: 리서치 & 자기검증 체이닝

## A. 구조화된 리서치

```xml
<research_question>
{{무엇을 알아내야 하는가}}
</research_question>

<success_criteria>
{{성공적인 답의 조건 — 무엇이 충족되면 끝인가}}
</success_criteria>

구조화된 방식으로 조사해라:
- 경쟁 가설 여러 개를 세우고, 진행노트에 각 신뢰도를 기록해라
- 여러 출처로 교차 검증해라
- 주기적으로 자기 접근법을 비판하고 계획을 갱신해라
- 가설 트리/리서치 노트 파일에 정보를 남겨 투명성을 확보해라
```

## B. 자기교정 체이닝 (GAN 원칙: 생성 ≠ 채점)

세 번의 분리된 호출로 품질을 2~3배 끌어올린다:

```
1단계 (생성)
{{초안 작성 지시}}

2단계 (채점) — 별도 호출, 다른 역할
아래 결과를 <criteria>에 비춰 냉정하게 평가하고 문제점을 등급(CRITICAL/HIGH/MEDIUM/LOW)으로 매겨라.
<criteria>{{평가 기준}}</criteria>
<draft>{{1단계 결과}}</draft>

3단계 (수정)
위 리뷰의 CRITICAL·HIGH를 모두 반영해 다시 작성해라.
```

> 빌더와 채점자를 분리해야 신뢰도가 산다. 자기 작업을 자기가 채점하면 후하게 준다.
