# 최신 모델 주의점 (Opus 4.x / Sonnet 4.6 / Fable 5)

> 핵심: 최신 모델은 **훨씬 능동적·적극적**이다. 예전 모델용으로 짠 "더 열심히 해" 류 프롬프트가
> 이제는 **과잉행동**(overtrigger)을 부른다. 압박을 빼는 방향으로 튜닝하라.

## 과잉 압박어를 빼라

```
❌  CRITICAL: 너는 반드시 이 도구를 써야 한다
✅  이 도구는 ~할 때 사용해
```
"반드시/MUST/무조건"으로 도구·스킬 호출을 강제하던 프롬프트는 이제 과하게 작동한다.

## thinking은 adaptive로

- Opus 4.x / Sonnet 4.6 = **adaptive thinking** (`thinking: {type: "adaptive"}`). effort로 깊이 조절.
- 구식 `budget_tokens` 방식은 deprecated → `effort` ("low/medium/high/xhigh/max") 또는 `max_tokens`로.
- 너무 많이 생각해서 느리면: "다중 단계 추론이 필요할 때만 thinking을 써라. 애매하면 바로 답해라."
- **결정 후 번복 금지** 유도: "접근법을 골랐으면 밀고 나가라. 모순되는 새 정보가 없으면 재검토하지 마라."

> 참고: thinking 꺼진 Opus 4.5는 "think"란 단어에 민감 → "consider / evaluate / reason through"로 대체.

## Prefill(어시스턴트 응답 미리 채우기) 폐지

Claude 4.6+ 부터 **마지막 어시스턴트 턴 prefill 미지원** (400 에러). 대안:

| 기존 prefill 용도 | 대안 |
|---|---|
| JSON/형식 강제 | **Structured Outputs** 기능 또는 그냥 스키마를 지시 |
| 서두("Here is...") 제거 | 시스템에 "서두 없이 바로 답하라" 명시 |
| 잘못된 거부 회피 | 최신 모델은 거부 판단이 좋아짐 → user 메시지로 충분 |
| 중단된 응답 이어쓰기 | user 턴에 "직전 응답이 `[...]`에서 끊겼어. 이어서 써" |

## 과잉설계(Overengineering) 억제

최신 Opus는 시키지 않은 파일·추상화·유연성을 더하는 경향. 최소화하려면:

```
오버엔지니어링 금지. 요청했거나 명백히 필요한 변경만 해라.
- 범위: 요청 외 기능 추가·리팩터·"개선" 금지
- 문서화: 안 건드린 코드에 주석·타입 추가 금지
- 방어코딩: 일어날 수 없는 시나리오의 예외처리 금지. 경계(입력/외부API)에서만 검증
- 추상화: 일회성 작업에 헬퍼/유틸 만들지 마라
```

## 서브에이전트 과용 주의

최신 모델은 알아서 위임을 잘 한다. 단 Opus 4.6은 단순 grep이면 될 일에도 서브에이전트를 남발.

```
서브에이전트는 병렬 실행·격리된 컨텍스트·독립 작업일 때만 써라.
단순 작업·순차 작업·단일 파일 편집은 직접 처리해라.
```

## 에이전트 안전 (위험 행동 확인)

가이드 없으면 되돌리기 힘든 행동(파일 삭제·force push·외부 발송)을 그냥 해버릴 수 있다.
→ "되돌리기 어렵거나 공유 시스템에 영향 주는 행동은 먼저 확인받아라" 명시.
(이건 내 글로벌 CLAUDE.md 안전 규칙과 동일한 방향)

## 멀티 컨텍스트 윈도우 작업 (긴 자동화)

- 첫 윈도우는 **셋업 전용** (테스트 작성, `init.sh` 등) → 이후 윈도우는 todo 반복.
- 상태는 **구조화**(테스트 결과 = JSON) + **자유서술**(진행노트 = txt) + **git 로그**로 추적.
- "토큰 부족하다고 일찍 멈추지 마라. 한계 근처면 상태를 저장하고 계속해라" 명시.
