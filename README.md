# 프롬프트 엔지니어링 정리

> Anthropic 공식 가이드를 **내 맥락에 맞게** 압축한 작업용 레퍼런스.
> 출처: [Prompting best practices](https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices)
> (구 URL `docs.claude.com/.../prompt-engineering/overview` → 자동 리다이렉트)

## 한 줄 결론

**내가 이미 쓰는 형식(명확한 지시 + 단계별 번호 + XML 태그 + 예시)이 그대로 "좋은 프롬프트"의 정석이다.**
시중 템플릿은 참고용으로 얼마든지 가져다 써도 된다 — 단, **빈칸에 내 맥락을 채우는 것**이 핵심이다.
템플릿을 그대로 복붙하면 "AI slop"이 나온다. 내 프로젝트의 성공 기준·제약·예시가 들어가야 결과가 산다.

## 시작하기 전 (Anthropic 전제 3가지)

프롬프트를 만지기 전에 이게 먼저다. 이게 없으면 프롬프트 튜닝은 헛돈다:

1. **성공 기준**을 명확히 정의했는가? ("이게 되면 성공이다")
2. 그 기준을 **경험적으로 테스트**할 방법이 있는가?
3. 개선할 **초안 프롬프트**가 있는가?

> 비용·지연시간 문제는 프롬프트가 아니라 **모델 교체**로 푸는 게 빠를 때가 많다.

## 쓰는 법

1. 새 작업이 생기면 → [templates/](templates/)에서 맞는 골격을 복사
